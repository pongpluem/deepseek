Preprocessing function จะมีหน้าตาแบบนี้ครับ:

EOS_TOKEN = tokenizer.eos_token  # Must add EOS_TOKEN


def formatting_prompts_func(examples):
    inputs = examples["Question"]
    cots = examples["Complex_CoT"]
    outputs = examples["Response"]
    texts = []
    for input, cot, output in zip(inputs, cots, outputs):
        text = train_prompt_style.format(input, cot, output) + EOS_TOKEN
        texts.append(text)
    return {
        "text": texts,
    }

ใช้ฟังก์ชั่น

dataset = dataset.map(formatting_prompts_func, batched = True,)

FastLanguageModel.for_inference(model)  # Unsloth has 2x faster inference!

แต่เดียวก่อนอะไร คือ GGUF model ล่ะ

GGUF Model จะเป็นรูปแล้วการ save model ที่ทำให้เราสามารถอ่าน และใช้ model ในการ inference ได้อย่างสะดวกครับ การใช้ GGUF model ทำให้เราสามารถใช้ model ได้โดยไม่ต้องติดตั้ง Python หรือ Huggingface แค่มี GGUF model เราก็สามารถเรียกใช้ได้เลย

สามารถอ่านเพิ่มเติมได้ที่ Link นี้

https://github.com/ggerganov/ggml/blob/master/docs/gguf.md

และแน่นอนว่า Ollama ก็มีการ Save model เป็น GGUF เช่นกันครับ ดังนั้นถ้าเราจำเป็นต้องแปลง model ของเราเป็น GGUF ก่อน

สร้าง Ollama model จาก GGUF ไฟล์

หลังจากที่เราโหลด model มาไว้บน Run time ของเราแล้วให้เรา Run save_pretrained_gguf เพื่อ save model เป็น GGUF บน local storage ครับ

model.save_pretrained_gguf(
    "/tmp/model", # บน kaggle Path /tmp/ จะเก็บข้อมูลได้มากกว่า 
    tokenizer,
    quantization_method="q4_k_s"
)

ให้สังเกตว่า ฟังก์ชั่นนี้จะมี quantizaiton_method ที่บอกระดับของ quantization นะครับ ขึ้นอันนี้คอมใครไหวไปก่อนเลย แต่ผมรันบน Kaggle ถ้าเอา q สูง ๆ บางที storage มันจะเกินครับ เลยเลือกเป็น q4_k_s ส่วน method อื่น ๆ สามารถดูได้ตามนี้เลย

[Optional] Upload ขึ้น Huggingface

อันนี้ไม่ต้องทำก็ได้นะครับ แต่สำหรับคนที่อยากจะอัพโหลด GUFF model ขึ้น Huggingface ด้วย ให้ใช้คำสั่งนี้ได้เลย แต่จะกินที่พอสมควร

# กินที่มากกกกกกก ไม่ควรรัน

Upload ขึ้น Ollama

เริ่ม แรกมาที่เว็บ Ollama.com สมัครสมาชิก แล้วกดที่รูปไอคอน แล้วไปที่ My models เป็นสร้าง model repo ใหม่ครับ อย่าลืมจำ Username ของตัวเองนะครับ

สร้าง Repository ของ model ขึ้นมาใหม่

ตั้งชื่อไฟล์ แล้วเลือกว่าจะให้เป็น Public หรือ Private

ทีนี้เราก็จะได้ model reposity ที่เราจะใช้ในการเก็บ model แล้วนะครับ ทีนี้กลับไปที่ Kaggle กัน

Modelfile:

อันนี้ Unsloth อ้างว่าสามารถ Generate Modelfile ได้เลย แต่ผมพบว่า unsloth บนเวอร์ชั่นที่รันบน Kaggle ไม่ยอม generate Modelfile ขึ้นมาครับ เลยต้องสร้างขึ้นมาเอง

MODELFILE = """FROM unsloth.Q4_K_S.gguf

TEMPLATE \"""Below is an instruction that describes a task, paired with an input that provides further context. 
Write a response that appropriately completes the request. 
Before answering, think carefully about the question and create a step-by-step chain of thoughts to ensure a logical and accurate response. {{ if .Prompt }}

### Instruction:
{{ .Prompt }}{{ end }}

### Response:
{{ .Response }}<|end_of_text|>\"""

PARAMETER stop "<|eom_id|>"
PARAMETER stop "<|python_tag|>"
PARAMETER stop "<|start_header_id|>"
PARAMETER stop "<|finetune_right_pad_id|>"
PARAMETER stop "<|eot_id|>"
PARAMETER stop "<|end_header_id|>"
PARAMETER stop "<|end_of_text|>"
PARAMETER stop "<｜end▁of▁sentence｜>"
PARAMETER stop "<|reserved_special_token_"
"""

with open("Modelfile", "w") as f:
    f.write(MODELFILE)

ตอนนี้ local file ของเราจะมี Modelfile กับ unsloth.Q4_K_S.gguf ครับ

ls
# Modelfile unsloth.Q4_K_S.gguf

ทีนี้เราก็รันคำสั่งเพื่อสร้าง ollama model

ollama create $OLLAMA_USERNAME/Deepseek-r1-medical-reasoning -f ./Modelfile

ถึงเวลา push ไปที่ Ollama ครับ

ขั้นตอนนี้เราจะทำการอ่าน Public key ที่ถูก set บน Ollama นะครับ อันนี้เราจะทำการ Push ด้วย Kaggle Public key จะถูกเปลี่ยนเสมอ

เราจะอ่าน Public key ด้วยคำสั่ง

cat ~/.ollama/id_ed25519.pub

เราก็จะ public key ที่จะต้องเอาไปใส่หน้า account ครับ

เข้าไปที่ link https://ollama.com/settings/keys แล้วกดเพิ่มคีย์ครับ

หลังจากที่ทำการเพิ่ม key เรียบร้อยแล้วให้ทำการ push model ขึ้นไปได้เลย

PUSH MODEL!

ollama push batprem/Deepseek-r1-medical-reasoning

เราก็จะได้ model ที่หน้า repository ของเราครับ

คราวนี้ลอง Pull model มาลองใช้ บนเครื่อง Mac

Note: Machine ที่ใช้ Run เป็น Mac M3 Pro Ram 18 GB นะครับ

รัน

ollama run <Ollama Repo name>
